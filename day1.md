## 기본 개념 설명

### 개발의 5대 핵심 요소

**프론트엔드(Front-end)** : 사용자가 웹사이트나 애플리케이션을 방문했을 때 눈에 보이고 직접 상호작용하는 모든 부분

  * 프론트엔드 언어, 3대 구성 요소: HTML, CSS, JS
    1. **HTML:** 웹페이지의 **뼈대** 와 구조를 잡습니다 (예: 제목, 문단, 버튼 배치).
    2. **CSS:** 웹페이지를 **디자인** 합니다 (예: 색상, 크기, 간격, 글꼴).
    3. **JavaScript:** 웹페이지에 **생동감** 을 불어넣습니다 (예: 클릭 시 알림창 띄우기, 실시간 데이터 업데이트).

**백엔드(Back-end)** : 웹사이트나 앱에서 '데이터 저장, 처리, 연산'을 담당하는 서버, API, DB 전체를 통칭합니다. 프론트엔드(화면)와 반대되는 개념

  * 백엔드의 핵심 기능:
    1. **서버(Server):** 사용자의 요청을 받아 무엇을 할지 결정하고 처리하는 '두뇌' 역할입니다.
    2. **애플리케이션 로직(API):** 데이터를 어떻게 가공하고 검증할지 정의하는 '규칙'입니다.
    3. **데이터베이스(Database):** 정보를 영구적으로 저장하는 '창고'입니다. (여기에 DB나 구글 시트가 해당됩니다.)

**배포(Deploy)** : 개발자가 내 컴퓨터(로컬)에서 만든 프로그램이나 웹사이트를 실제 사용자들이 접속해서 사용할 수 있도록 서버(인터넷 세상)에 올리는 과정

  * **배포가 필요한 이유:** 내 컴퓨터에서 만든 웹사이트는 내 컴퓨터 주소(예: `localhost:3000`)로만 접속이 가능합니다. 다른 사람들이 접속하려면 24시간 켜져 있고 인터넷에 연결된 **공용 서버** 로 코드를 옮겨야 합니다.
  * **배포의 주요 단계:**
    1. **빌드(Build):** 작성한 코드의 용량을 줄이고, 브라우저나 서버가 읽기 좋은 상태로 변환합니다.
    2. **테스트(Test):** 배포 전 코드에 오류가 없는지 자동으로 검사합니다.
    3. **릴리스(Release):** 테스트를 통과한 결과물에 버전을 붙여 준비합니다.
    4. **전파(Deployment):** 준비된 결과물을 클라우드 서비스(AWS, Vercel, Google Cloud 등)에 업로드하여 활성화합니다.

## LLM으로 대시보드 만들기 실습

### 단계별 프롬프트로 만들기 실습

**[Step 1] 역할 + 목적 프롬프트**

  * 너는 10년 차 HR 매니저를 돕는 AI 서비스 기획자야.
  * 나는 직관적인 웹 대시보드를 보고 싶어.
  * 목적은 '조직 현황을 한눈에 파악하는 것'이야.

**[Step 2] 화면 구조 생성**

  * 웹 화면 형태로 만들어줘.
  * 왼쪽에는 고정된 사이드바,
  * 상단에는 핵심 지표를 보여주는 카드 3개,
  * 가운데에는 목록이나 차트가 보이게 해줘.
  * 아직 실제 데이터는 없어도 돼.

**[Step 3] 디자인 조건 주기**

  * 전체 톤은 신뢰감을 주는 비즈니스 스타일로.
  * 컬러는 Deep Navy와 White 위주로 사용해줘.
  * 과도한 애니메이션은 필요 없어.

**[Step 4] 데이터 추가**

  * 한국 조직을 가정한 가상의 직원 데이터 20개를 만들어줘.
  * 이름, 부서, 직급, 입사일, 재직상태를 포함해.
  * 이 데이터를 화면에 리스트 형태로 보여줘.

**[Step 5] 사고의 확장**

  * 리스트 상단에 이름 검색창과 부서별 필터 드롭다운을 추가해줘.
  * 입력하면 실시간으로 반영되게 해줘.

**[Step 6] 비즈니스 로직 추가하기**

  * 간단한 분석 패널을 추가해줘.
  * 기준:
    * 입사 1년 미만
    * 잔여 연차 0
    * 이 조건에 해당하면 '주의 필요'로 표시해줘.

**[Step 7] 수정·개선 프롬프트**

  * 구조는 좋은데, 글자가 작아서 가독성이 떨어져.
  * 카드와 영역에 약한 그림자를 넣고, 전체 폰트를 조금 키워줘.

**[Step 8] 수정·개선 프롬프트**

  * 방금 만든 구조를 '내 업무(예시: 재고관리 업무)'에 맞게 바꿔줘.
  * 화면 구조는 유지하되, 데이터와 지표만 변경해줘.

#### 코드 수정하기

**P:** 종합 대시보드에서 인력 분석, 채용 파이프라인, 운영 효율 메뉴를 선택하면 메뉴와 화면 이동이 되어야 합니다. 각 메뉴 선택시 상세 내용이 나타나야 합니다.:

  * **종합 대시보드:**
    * Top (KPI Cards): 총 인원수, 평균 연봉, 이번 달 채용 합격자, 미처리 업무 건수.
    * Middle: 부서별 인원 분포(Pie Chart)와 부서별 평균 연봉 비교(Bar Chart).
    * Bottom: 최근 발생한 주요 업무 요청 리스트 (상위 5건).
  * **인력 분석:**
    * 조직 구성: 직급별 인원 구성 및 성비(데이터 있을 시).
    * 조직 건강도: 만족도 vs 이직률 스캐터 플롯(Scatter Plot).
    * 기획 의도: 만족도는 낮으나 이직률이 낮은 부서 등을 찾아내어 인사 사고 예방.
    * 근속 분석: 부서별 평균 근속연수 추이 시각화.
  * **채용 파이프라인:**
    * 채용 퍼널: 지원자 → 서류 → 면접 → 최종합격 → 입사 (Funnel Chart).
    * 효율성 지표: 월별 지원자 추이(Line Chart) 및 최종 입사율 계산.
    * 채용 속도: 요청 대비 채용 완료까지 소요 기간(Data 계산 필요).
  * **운영 효율성:**
    * 업무 부하: 담당자별(강미라, 오태현 등) 처리 중인 업무 건수 히트맵.
    * 요청 유형: 휴가, IT지원, 경비 등 유형별 비중(Donut Chart).

### Add Assets(자산 추가)

Opal에서 **'Add Assets(자산 추가)'** 기능은 AI가 결과물(앱, 콘텐츠, 답변 등)을 생성할 때 **참고해야 할 '재료'나 '지식'을 제공하는 곳** 입니다.

단순히 파일을 업로드하는 것을 넘어, **AI에게 "이 데이터를 기반으로 생각하고 작업해"라고 맥락(Context)을 부여** 하는 핵심 단계입니다.

#### 각 항목별 상세 기능

  * **Upload file (파일 업로드):** 사용자의 컴퓨터에 있는 문서(PDF, DOCX), 이미지, 오디오 파일 등을 직접 올립니다. (예: 회사 소개서, 로고 이미지, 회의 녹음 파일 등)
  * **My Drive (구글 드라이브):** 구글 드라이브에 저장된 문서를 바로 연동합니다. 클라우드에 있는 자료를 다운로드 없이 바로 AI에게 학습시킬 수 있어 효율적입니다.
  * **YouTube (유튜브):** 유튜브 영상의 링크를 넣으면, AI가 해당 영상의 내용(스크립트, 주제 등)을 분석하여 참고 자료로 활용합니다.
  * **Text (텍스트):** 별도의 파일 없이 사용자가 직접 지시 사항이나 참고할 텍스트를 입력하거나 붙여넣기(Copy & Paste) 할 때 사용합니다.
  * **Drawing (드로잉):** 간단한 스케치나 아이디어를 직접 그려서 시각적인 가이드를 제공합니다. (예: 앱의 대략적인 레이아웃 스케치)

#### 왜 중요한가요? (핵심 가치)

이 기능이 중요한 이유는 **'나만의 맞춤형 결과물'**을 만들기 위해서입니다.

  * **Generic (일반적) vs. Personalized (개별화):**
    * Assets 없이 질문하면: AI가 인터넷에 있는 일반적인 지식을 바탕으로 답변합니다.
    * **Assets을 추가하면:** AI가 사용자가 제공한 **'우리 회사 규정', '나의 메모', '특정 유튜브 강의'** 내용을 바탕으로 분석하고 결과물을 만듭니다.

#### 활용 예시

만약 **"매일 아침 루틴을 관리하는 앱"**을 만든다고 가정해 보겠습니다.

  1. **Text:** "아침 6시 기상, 7시 조깅, 8시 독서"라는 텍스트를 입력합니다.
  2. **Upload file:** 평소에 읽던 '건강 관리 가이드 PDF'를 업로드합니다.
  3. **Generate:** "이 자료들을 바탕으로 내 아침 루틴을 체크하고 조언해 주는 앱을 만들어 줘"라고 요청합니다.

결과적으로 Opal은 선생님이 제공한 **Assets(자산)**을 분석하여 단순한 앱이 아닌, **선생님의 생활 패턴과 건강 지식이 반영된 맞춤형 앱** 을 생성하게 됩니다.
